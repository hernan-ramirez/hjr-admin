Overview

> About WebTabs

WebTabs is a multi-row DHTML tab widget that was created out of the need to have a fast-responding, multi-row tab widget that emulates the look and feel of the standard Windows tab widget.

We use this widget in our own outsource products, as a part of our web-based administration interfaces.

Different editions of WebTabs available with varying features and usage rights, for different uses. Please check the Webtabs product home page for more information.

Features

> Multi-row tabs

Never again be lmited to a single row of tabs! Multiple rows of tabs are supported.

> Tab icons

You can include 16x16 pixel icon images to display next to tab headings. You may select which tabs (none, some or all) have icons.

> Instant client-side page switching

All tab pages that hold the content for your tabs are loaded automatically at start-up, so clicking and activating a tab displays its contents instantly. Tab-switch rendering is performed by JavaScript on the client browser, so no slow server-fetches are required. Users no longer have to experience long delays when selecting a tab.

> Vector graphics rendering

The actual drawing of the widget is performed using DIV elements, providing pixel-perfect control.

> No slow-loading rendering images used

The vector rendering used draws all round corners, so no corner or tab images are used as is often the case with many other HTML tab widgets.

> Automatic single-row/multi-row detection and tab resizing

Multi-row tabs are automatically used if your tabs do not fit on one row. You don't have to worry about managing any tabs at all.

> Cross-browser platform support

Cross-browser support means that the higest number of people possible can view your tab widgets, whether you are building web sites or DHTML applications.

Widget display is independent on the underlying operating system, so Windows, Macintosh and UNIX/Linux users can all access your tabs using a supported browser.

Currently supported browsers:

  • Internet Explorer 5.0 +
  • Netscape 6.0 +
  • Mozilla 1.0 +

    > Theme customization through CSS

    You can easily change the colours used for the tab header text, background and borders.

    The default style sheets automatically use the colours from the user's own desktop settings.

    > No Java required

    Because the widget is written in JavaScript, you don't have to worry about Java being installed on client computers.

    Tutorial

    This tutorial will show you how to get your tab project up and running quickly.

    The easiest way to create your tab widget is to use one of the supplied demo files as a template, and to build from there. The "Live Demo" page shows these files in use.

    To build your WebTabs tab widget from scratch, follow these instructions.

    > Setup

    You need the files "webtabs.js", which contains the WebTabs JavaScript source code, and "webtabs.css", which contains the WebTabs theme style sheet declarations.

    Put these files in the same directory as your new HTML page.

    > The Head Code

    The following code must be placed in the HEAD section of your page.

    1. Create a link to include the style sheet file "webtabs.css".
    <link type="text/css" rel="stylesheet" href="webtabs.css"></link>

    2. Create a link to include the script file "webtabs.js".
    <script type="text/javascript" src="webtabs.js"></script>

    3. Create a script containing a function that defines and initializes the WebTab object.
    <script language=javascript>
      function f_start() {
        var Widget = new WebTabs_widget(200, 100, 10, 10, "absolute")
        Widget.add(new WebTabs_tab("Title 1", "page_1", null))
        document.getElementById("WebTabs_container").innerHTML = Widget
        Widget.f_init_tabs()
      }
    </script>
  • Replace the "200" and "100" with the pixel width and height of the widget, respectively.
  • Replace the "10" and "10" with the X and Y pixel coordinates of the widget, respectively.
  • If you want other HTML content before or after the widget, replace "absolute" with "relative".
  • Replace the tab heading text "Title 1" with the text of your choice for the first tab.
  • If you want to display an icon image file in this tab, change the "null" to the image file name.
  • Copy the "Widget.add" line for as many tabs as you want, and change the text/page/icon.

    > The Body Code

    The following code must be placed in the BODY section of your page.

    4. Create the DIV element that contains your pre-widget HTML content (if any).
    <!--
    <div style="position: relative">
      content before widget
    </div>
    -->
  • If you want to display content before the widget, remove the comments and fill in the content.
  • If you want the widget to display relative to any pre-widget content, set the WebTabs_widget objet position to "relative" and the "X, Y" coordinates to "0, 0".

    5. Create the DIV element that will be the container for the widget.
    <div id=WebTabs_container></div>

    6. Create the DIV element that contains your post-widget HTML content (if any).
    <!--
    <div style="position: relative">
      content after widget
    </div>
    -->
  • If you want to display content after the widget, remove the comments and fill in the content.

    7. Create the DIV page elements that will contain your individual tab content.
    <div id=page_1 class=WebTabs-external-page-container>
      <div class=WebTabs-internal-page-container>
        My page 1
      </div>
    </div>
  • Copy the page container lines for as many tabs as you have defined, and increase the page number.

    8. Finally, create a script that runs the function defined in the HEAD section, to initialize the widget.
    <script>f_start()</script>
  • Note: Running the function from the BODY tag's ONLOAD event handler is not cross-browser compatible.

    > Caveats

  • Displaying multiple tab widgets per page.

    It is only possible to create one tab widget per window. If you want to display multiple widget instances on a page, in one window, you must load each widget in a separate IFRAME element. This effectively uses a window within a window. An example of this technique being used is visible on the "Live Demo" page of this tutorial.
    • API Reference

    > The WebTabs_widget Object

    Constructor Syntax:

    new WebTabs_widget(w, h, x, y, position)

    Contructor Arguements:

    Name Type Description
    w integer The width of the tab widget in pixels.
    h integer The height of the tab widget in pixels.
    x integer The X coordinate of the tab widget's top left corner in pixels.
    y integer The Y coordinate of the tab widget's top left corner in pixels.
    position string The position of the tab widget.
  • Has to be one of "absolute" or "relative".
  • If "absolute", then the position is relative to the parent container; usually the current window.
  • If "relative", then the position is relative to the HTML content in the page placed before the "WebTabs_container" DIV element.

    • Constructor Usage Example:

    var Widget = new WebTabs_widget(250, 140, 10, 10, "absolute")

    Methods:

    Name Returns Description
    add(tab) void Adds a WebTabs_tab object to the WebTabs_widget object (tab widget).
    f_init_tabs() void Initializes the tab widget.
    f_activate_tab(i) void Activates tab number i, where numbering starts at 0.
    f_move_to(x, y) void Move tab widget to X and Y coordinates as indicated in pixels.
    f_move_by(x, y) void Move tab widget by X and Y coordinates as indicated in pixels..

    > The WebTabs_tab Object

    Constructor Syntax:

    new WebTabs_tab(text, page, icon)

    Contructor Arguements:

    Name Type Description
    text string The title text of the tab.
    page string The name of the DIV element containing the HTML onctnet of the tab.
    icon string The name of the icon image file to use for the tab, or a null string ("", '' or null) if no icon is to be used.
  • The image dimensions should be 16x16 pixels.

    • Constructor Usage Example:

    Widget.add(new WebTabs_tab("Home", "page_1", "home.gif"))
    or
    Widget.add(new WebTabs_tab("Home", "page_1", null))
